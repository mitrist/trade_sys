# Стартовый промпт для запуска мультиагентного цикла

Скопируй блок ниже в чат Cursor, подставь свой план вместо примера и отправь. Убедись, что включено правило **multi-agent-dev-flow** (или что в контексте есть описание процесса из `.cursor/rules/multi-agent-dev-flow.mdc`).

---

**Текст для вставки:**

```
Запусти мультиагентный цикл разработки (Dispatcher → Implement → QA) по следующему глобальному плану.

План:
1. [Первая задача — опиши конкретно]
2. [Вторая задача]
3. [Третья задача]
…

Правила:
- Ты по очереди выполняешь роли Dispatcher, Implement и QA.
- После каждой реализации (Implement) проверяй код по чек-листу QA (синтаксис, логика, безопасность, производительность, стиль, обработка ошибок) и возвращай вердикт в формате {"verdict": "OK"} или {"verdict": "NOT_OK", "comment": "…"}.
- При "НЕ ОК" передай задачу обратно Implement на доработку; при "ОК" переходи к следующему пункту плана.
- По завершении всех пунктов сообщи о завершении и перечисли выполненные задачи и изменённые файлы.
```

---

**Пример с конкретным планом:**

```
Запусти мультиагентный цикл разработки (Dispatcher → Implement → QA) по следующему глобальному плану.

План:
1. Создать файл config.py с настройками приложения (DEBUG, SECRET_KEY из env).
2. Написать функцию get_connection() в db.py для подключения к SQLite.
3. Написать тесты для get_connection() в tests/test_db.py.

Правила:
- Ты по очереди выполняешь роли Dispatcher, Implement и QA.
- После каждой реализации проверяй код по чек-листу QA и возвращай вердикт: {"verdict": "OK"} или {"verdict": "NOT_OK", "comment": "…"}.
- При "НЕ ОК" отдай задачу обратно Implement на доработку; при "ОК" — следующая задача.
- В конце сообщи о завершении и перечисли задачи и файлы.
```
